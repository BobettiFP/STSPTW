## TSP-TW Torch Datasets (Run `2026-02-27_15-19-06`)

This document describes the **PyTorch-formatted TSP with Time Windows (TSP-TW)** datasets:

- `tsp_tw_10.pt`
- `tsp_tw_30.pt`
- `tsp_tw_100.pt`

All files live under `vrp_bench/data/tsp_tw/2026-02-27_15-19-06/` and are generated by `generate_tsp_tw_instances.py`, then saved via `save_dataset_torch`.

---

## Common structure (all `.pt` files)

Each `.pt` file is a `dict` saved with `torch.save`, intended to be loaded using `load_dataset_torch`. The structure is identical across the three files; only the **number of customers per instance** changes.

Let:

- `N` = number of instances per file (here `N = 10`, from `NUM_INSTANCES` in `constants.py`).
- `n_nodes` = `1 + num_customers` = number of nodes (1 depot + customers).

After loading, each file contains the following keys:

- **`locations`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N, n_nodes, 2)`
  - **Meaning**: 2D coordinates \((x, y)\) of each node.
    - Index `0` is the **depot**.
    - Indices `1..n_nodes-1` are **customers**.
  - **Domain**: Points lie on a rectangular map of size `map_size` (typically `[0, 1000] Ã— [0, 1000]`).

- **`demands`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N, n_nodes)`
  - **Meaning**: Per-node demands in a VRP-style format.
    - Depot demand is `0`.
    - Customer demands are `1`.
  - For TSP-TW, capacity is effectively **non-binding** due to large vehicle capacity.

- **`time_matrix`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N, n_nodes, n_nodes)`
  - **Meaning**: Dense matrix of travel times (minutes) between all node pairs.
    - Entry `[i, j]` is the (expected) travel time from node `i` to node `j`.
    - Diagonal entries `[i, i]` are `0.0`.
  - Derived from the underlying `travel_times` dict using a stochastic travel-time model (distance, speed, and delay).

- **`time_windows`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N, n_nodes, 2)`
  - **Meaning**: Hard time windows per node, in minutes from midnight.
    - For node `k`, `[..., k, 0]` is the earliest service start time.
    - `[..., k, 1]` is the latest allowable service time.
  - **Depot (node 0)**: time window fixed to `(0, 1440)` (full day).
  - **Customers**: time windows sampled using the paperâ€™s residential/commercial ratio (â‰ˆ60% residential, 40% commercial).

- **`appear_times`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N, n_nodes)`
  - **Meaning**: Earliest appearance time of each node (minutes from midnight).
  - For these TSP-TW datasets, generation uses `is_dynamic=False`, so `appear_times` is expected to be all zeros (static case).

- **`num_vehicles`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N,)`
  - **Meaning**: Number of vehicles per instance.
  - For TSP-TW: always `1` (single-vehicle travelling salesman).

- **`vehicle_capacities`**
  - **Type**: `torch.Tensor`
  - **Shape**: `(N,)`
  - **Meaning**: Vehicle capacity for each instance.
  - Value: `1e9` (very large), effectively disabling capacity constraints.

- **`travel_times`**
  - **Type**: `list` of length `N`
  - **Per-instance element**: `Dict[Tuple[int, int], float]`
    - Keys: `(i, j)` index pairs.
    - Values: travel times (minutes) from `i` to `j`.
  - Meaning: Original sparse representation used during generation; `time_matrix` is its dense counterpart.

- **`map_size`**
  - **Type**: Python list or tuple, e.g. `[1000, 1000]`
  - **Meaning**: Map width and height for node coordinate generation.

- **`num_cities`**
  - **Type**: `int`
  - **Meaning**: Number of city clusters used to generate locations.
  - Typically `max(1, num_customers // 50)`, influencing spatial clustering.

- **`num_depots`**
  - **Type**: `int`
  - **Meaning**: Number of depots in the instance.
  - For TSP-TW: always `1` (depot index `0`).

## File-specific details

### `tsp_tw_10.pt`

- **Path**: `vrp_bench/data/tsp_tw/2026-02-27_15-19-06/tsp_tw_10.pt`
- **Customers per instance**: `num_customers = 10`
- **Nodes per instance**: `n_nodes = 1 (depot) + 10 (customers) = 11`
- **Instances per file**: `N = 10`
- **Key shapes**:
  - `locations`: `(10, 11, 2)`
  - `demands`: `(10, 11)`
  - `time_matrix`: `(10, 11, 11)`
  - `time_windows`: `(10, 11, 2)`
  - `appear_times`: `(10, 11)`
  - `num_vehicles`: `(10,)`
  - `vehicle_capacities`: `(10,)`
- **Use cases**:
  - Very small benchmark: ideal for debugging solvers, unit tests, and quick experiments.

## Loading examples

You can load and inspect these datasets as follows:

```python
from vrp_bench.common import load_dataset_torch

data_10 = load_dataset_torch("vrp_bench/data/tsp_tw/2026-02-27_15-19-06/tsp_tw_10.pt")
data_30 = load_dataset_torch("vrp_bench/data/tsp_tw/2026-02-27_15-19-06/tsp_tw_30.pt")
data_100 = load_dataset_torch("vrp_bench/data/tsp_tw/2026-02-27_15-19-06/tsp_tw_100.pt")

print(data_10["locations"].shape)      # (10, 11, 2)
print(data_30["time_matrix"].shape)    # (10, 31, 31)
print(data_100["time_windows"].shape)  # (10, 101, 2)
```
